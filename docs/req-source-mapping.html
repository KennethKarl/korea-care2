구조 설명. 원본 DB는 한 화면을 영역구분별 여러 행으로 쪼개 두었고, 과거 드래프트 중복까지 섞여 총 107개 행입니다(27개 최신 통합행 + 80개 레거시/섹션). SRS는 이를 화면 1개 = REQ 1행으로 통합한 43개 행입니다. 따라서 다수 원본 행 → 1개 REQ로 합쳐집니다(1:N 통합). 아래는 원본 모든 행이 어느 REQ로 들어갔는지를 화면별로 묶은 것입니다.
